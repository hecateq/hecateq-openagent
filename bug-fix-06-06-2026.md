## Implementation Details

### 1. Agent Index Advisory/Runtime Truth Alignment

**Files:** `src/tools/delegate-task/subagent-resolver.ts` (JSDoc block)

Added a comprehensive JSDoc contract clarifying the agent index's role as advisory/enrichment data that never overrides live runtime resolution. Five ordered precedence layers are documented:
1. Live registered agents (`client.app.agents()`)
2. Discovered custom agents (file-based discovery)
3. Config filters (`disabled_agents`, overrides)
4. Delegate-task resolution (`subagent-discovery.ts`)
5. Tool registry wiring (`tool-config-handler.ts`)

The agent index (`buildAgentIndexAdvisory`) is consumed only for suggestion ranking and confidence metadata. Resolution status (`exact_agent_found` / `exact_agent_disabled` / `exact_agent_unknown` / `category_fallback`) is always determined from live runtime candidates. Three invariants are explicitly documented: unknown exact agent â†’ hard error, disabled exact agent â†’ explicit disabled error, category fallback only when no subagent_type is given.

The `buildAgentIndexAdvisory` function's JSDoc was also clarified to document the strict advisory-only contract.

### 3. Context Injection Budget Visibility

**Files:** `src/config/schema/hecateq.ts`, `src/hooks/hecateq-project-context-injector/index.ts`, `src/config/schema.test.ts`, `src/plugin-config.test.ts`, `src/hooks/hecateq-project-context-injector/index.test.ts`, `assets/hecateq-openagent.schema.json`, `assets/oh-my-opencode.schema.json`

Added a new config field `include_budget_summary` (default `true`) to the context injection schema. When enabled, the context injector prepends a `<hecateq-context-budget>` XML tag to each rendered context block, reporting:
- Total character count per section (`memory`, `contracts`, `task_graphs`, `agent_index`, `git_state`, `overhead`)
- Overall `total` vs `max` characters
- Truncation flag

Three render paths (compact with root contract, compact without, expanded) all receive the budget tag. New `HecateqContextBudgetSection`, `HecateqContextBudgetReport` types and `computeContextBudgetReport()` / `renderBudgetSummaryCompact()` / `measureContextSectionLengths()` functions were added. Schema tests verify the new field defaults and output shapes.

### 4. Handoff + MEMORY_UPDATE Parser Hardening

**Files (handoff):** `src/features/hecateq-orchestration/handoff-parser.ts`
- **Null/empty guard:** `parseHandoffBlock()` now returns a structured result with `validationIssues` for `null`/`undefined`/non-string input instead of crashing
- **Signal object validation:** Each item in `SIGNALS_EMITTED` is validated â€” non-object entries are skipped with errors, missing `signal` fields get errors, unknown signal names (not in `KNOWN_SIGNAL_NAMES`) get warnings
- **Handoff target validation:** Unknown targets that aren't known routing directives or agent IDs produce a warning
- **`KNOWN_SIGNAL_NAMES` registry:** A local mirror of the DAG signal registry with 9 known signals, used to detect drift

**Files (MEMORY_UPDATE):** `src/shared/memory-update-signal.ts`
- **Markdown fence stripping:** `MEMORY_UPDATE` blocks wrapped in ```json...``` fences are now automatically unwrapped before JSON parsing
- **`QuarantinedBlock` interface:** Malformed blocks are recorded with snippet (max 200 chars), reason, and approximate line number
- **`issues` field:** Top-level parse issues are propagated in `MemoryUpdateParseResult`
- **Granular error reasons:** Distinguishes between empty blocks, malformed JSON inside fence, malformed JSON without fence, non-object JSON (array/primitive), and non-array `entries` field
- The previous `malformedBlocks` counter is still maintained for backward compatibility

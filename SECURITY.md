# Security Policy

## Supported Versions

| Version           | Supported          |
| ----------------- | ------------------ |
| 0.1.x (beta)      | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in Hecateq OpenAgent, please report it privately by opening a security advisory on GitHub:

**https://github.com/hecateq/hecateq-openagent/security/advisories/new**

Please do **not** report security vulnerabilities via public GitHub issues.

### What to include

- Description of the vulnerability
- Steps to reproduce (if applicable)
- Potential impact
- Any suggested fix (if known)

### Response timeline

We will acknowledge receipt within 72 hours and provide an initial assessment within 5 business days. We will keep you informed of progress toward a fix.

## Scope

This security policy covers the `@hecateq/openagent` npm package and its source code. Upstream vulnerabilities in `oh-my-openagent` or its dependencies should be reported to the respective projects.

## Best Practices

- Keep your installation up to date
- Review the [SUL-1.0 license](./LICENSE.md) for permitted use
- Report suspected vulnerabilities privately as described above

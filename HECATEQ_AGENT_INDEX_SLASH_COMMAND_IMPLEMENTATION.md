HECATEQ_AGENT_INDEX_SLASH_COMMAND_IMPLEMENTATION

## Scope

Add a Hecateq slash command that generates a global custom-agent capability index from global OpenCode agent markdown files, without introducing a manual JSON maintenance workflow.

This phase includes:
- slash command registration
- deterministic global agent indexing
- generated file write with overwrite safety
- doctor checks for missing/stale/weak/duplicate index state

This phase does not include:
- route resolver logic
- context injector consumption of the index
- project-specific Hecateq agent config
- AI-only indexing that can fail closed

## What Changed

- Added a new built-in slash command: `/hecateq-agent-index`
- Added a deterministic shared indexer: `src/shared/hecateq-agent-indexer.ts`
- Added direct slash-command execution support in `auto-slash-command` executor for this command
- Added generated JSON write logic for `~/.config/opencode/hecateq/agent-index.generated.json`
- Added overwrite safety to protect non-generated files
- Added doctor checks for index missing, invalid/unsupported version, stale state, weak metadata, and duplicates
- Added focused tests for the indexer, command execution, command registration, and doctor integration

## Slash Command

Slash command name: **`/hecateq-agent-index`**

Registration location:
- `src/features/builtin-commands/types.ts`
- `src/config/schema/commands.ts`
- `src/features/builtin-commands/commands.ts`

Execution path:
- discovered by the existing slash-command discovery system
- executed through `src/hooks/auto-slash-command/executor.ts`
- this command uses a deterministic runtime path instead of only injecting prompt text

## Agent Sources

Global agent path is resolved from the existing OpenCode config-dir helpers:
- `getOpenCodeConfigDirs({ binary: "opencode" })`

Scanned source of truth:
- `<opencode-config-dir>/agents/*.md`

Project-specific agents are used: **No**

This implementation intentionally avoids:
- `.opencode/agents`
- project-local Hecateq JSON
- manually curated agent capability JSON

## Generated Index Path

Generated output path:
- `~/.config/opencode/hecateq/agent-index.generated.json`

Runtime helper resolves this through:
- `getOpenCodeConfigDir({ binary: "opencode" })`

The file is generated/cache output and is not intended for manual editing.

## Index Schema

Defined in:
- `src/shared/hecateq-agent-indexer.ts`

Top-level shape:
- `version`
- `generated_at`
- `generator`
- `notice`
- `enrichment_mode`
- `source.agents_dirs`
- `summary.agents_discovered`
- `summary.agents_indexed`
- `summary.weak_metadata`
- `summary.duplicates`
- `agents[]`

Each agent entry includes:
- `name`
- `display_name`
- `filename`
- `source_file`
- `description`
- `body_preview`
- `role`
- `domains`
- `keywords`
- `use_when`
- `avoid_when`
- `confidence`
- `signals.filename`
- `signals.frontmatter`
- `signals.body`
- `warnings`

## Capability Extraction

Current behavior: **deterministic**

Role/domain/keyword extraction uses:
- filename tokens
- frontmatter `name` and `description`
- markdown body content
- deterministic domain dictionary scoring
- markdown section extraction for `When to use` and `Avoid when` style headings

Domain extraction currently covers starter buckets like:
- backend
- frontend
- flutter
- security
- qa
- devops
- docs
- database
- realtime
- ai
- scraping

## AI vs Deterministic Behavior

Current implementation: **deterministic**

Hybrid/AI enrichment status:
- AI enrichment is **future-phase ready**, but not wired in this phase
- command output explicitly notes that the current index is deterministic today

This choice keeps the command reliable and non-brittle while still removing the manual JSON burden.

## Overwrite Safety

Existing generated file overwrite behavior:
- **Yes**, existing generated file can be overwritten

Non-generated existing file overwrite behavior:
- **No**, overwrite is refused

Safety rule:
- existing file is overwriteable only if it already looks generated by this helper (`generator` or `notice` match)
- otherwise the command returns a refusal summary and leaves the file untouched

## Behavior Before

- No `/hecateq-agent-index` slash command existed
- No generated global custom-agent capability index existed
- User would still need a manual capability-mapping path outside runtime support
- Doctor could not verify generated index health

## Behavior After

- `/hecateq-agent-index` exists and is discoverable as a built-in command
- Running the command deterministically scans global OpenCode custom agent markdown files
- A generated capability index is written to `~/.config/opencode/hecateq/agent-index.generated.json`
- Non-generated files at that path are protected from overwrite
- Doctor can validate generated index health and freshness

## Risks

- Domain classification is heuristic and deterministic, so uncommon agent descriptions may still land in weak metadata
- AI enrichment is intentionally deferred, so role/use_when/avoid_when can be less rich than a future model-assisted pass
- The command currently uses direct executor-side runtime behavior for one command; future command-side effects should follow the same explicit pattern rather than leaking into generic template execution

## Rollback

1. Remove `src/shared/hecateq-agent-indexer.ts`
2. Remove `src/shared/hecateq-agent-indexer.test.ts`
3. Remove the direct execution branch for `hecateq-agent-index` from `src/hooks/auto-slash-command/executor.ts`

   - `src/features/builtin-commands/templates/hecateq-agent-index.ts`
   - `src/features/builtin-commands/commands.ts`
   - `src/features/builtin-commands/types.ts`

5. Remove the doctor checks and doctor tests for the generated index
6. Re-run the same test set
